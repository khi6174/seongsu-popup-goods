# 05 · 사용자 시나리오 (User Scenarios)  — v1.0

> 기획 산출물의 완결편. 컨셉(`01`)·화면(`02`)·데이터(`03`)·IA(`04`)를 하나의
> **실제 사용 이야기**로 꿰맨다. 각 단계에 **화면 ID(S1~S6, G1~G2)** 와 **데이터
> (Popup/Product/Cart/Order)** 를 붙여, 이 문서가 곧 개발·테스트의 대본이 되게 한다.
>
> **화면 ID 복습**: S1 홈 · S2 팝업상세 · S3 상품상세 · S4 장바구니 ·
> S5 체크아웃 · S6 주문완료 · G1 AI챗봇 · G2 언어/통화 전환.

## 페르소나 (누가 쓰는가)

### P1 · Emma (1순위 타겟) 🇺🇸→🇰🇷
- 27세, 미국인 관광객. K-컬처 팬으로 5일간 서울 여행 중, 오늘 성수동 방문 예정.
- 한국어를 못 읽는다. 캐리어가 꽉 차 무거운 짐은 부담. 스마트폰으로 모든 걸 한다.
- **원하는 것**: "지금 성수동에서 갈 만한 팝업"과 "가볍게 살 만한 굿즈"를 영어로.

### P2 · 지원 (2순위 타겟) 🇰🇷
- 24세, 국내 트렌드세터. 특정 캐릭터 팝업의 **키링을 콕 집어** 사고 싶다.
- 현장은 이미 다녀왔지만 원하던 게 품절이었다. SNS 인증을 즐긴다.

### P3 · Kenji (원격 구매자) 🇯🇵
- 31세, 일본 거주. 한국에 못 오지만 성수동 팝업 굿즈를 갖고 싶다.
- **해외배송 가능 여부와 총액(배송비 포함)** 이 가장 궁금하다.

## 시나리오 A — Emma의 메인 여정 (해피패스: 챗봇 + 둘러보기)
> 대화형으로 진입 → 둘러보기로 확신 → 비회원 주문까지. 이 몰의 대표 흐름.

| # | 사용자 행동 | 화면 | 시스템 / AI 반응 | 관련 데이터 |
|---|------------|------|-----------------|------------|
| 1 | 사이트 첫 진입. 화면이 한국어라 당황 | S1 | 헤더의 **🌐 KO/EN** 를 발견 | i18n |
| 2 | **EN** 클릭 | G2 | 전체 UI가 영어로 전환 | i18n(en) |
| 3 | 우하단 챗봇 버튼 클릭, "Any good popups in Seongsu right now?" | G1 | AI가 **status=OPEN 팝업** 3개를 이유와 함께 추천 + 링크 | Popup(status 계산) |
| 4 | 추천 중 하나 클릭 | S2 | 팝업 상세: 영문 소개 + 소속 굿즈 목록 | Popup, Product |
| 5 | 가벼운 **키링**에 눈이 감 → 상품 클릭 | S3 | 상품 상세: 이미지 + **영어 설명(자동번역)** + `₩12,000 ≈ $9` | Product(desc_en), 환율 |
| 6 | "Add to Cart" | S3→S4 | 장바구니에 담김, 헤더 🛒 배지 +1 | CartItem |
| 7 | 스티커도 추가로 담음 | S3→S4 | 수량/합계 갱신 | CartItem |
| 8 | 🛒 클릭, 장바구니 확인 | S4 | 상품 합계(subtotal) 표시 | CartItem |
| 9 | "Checkout" | S5 | 이름·이메일·**배송 국가(USA)** 입력 폼 | Order 초안 |
| 10 | 국가 선택 → 배송비 자동 계산 | S5 | region=`OTHER` → 고정 배송비 + **총액(₩ + ≈$)** | shipping_region, 환율 |
| 11 | "Place Order"(모의결제) | S5→S6 | 주문 생성, `status=CREATED` | Order, OrderItem(스냅샷) |
| 12 | 주문번호 확인 | S6 | `SS-20260702-0001` + 요약 표시 | Order |

**결과**: 언어 장벽 없이, 줄·짐 없이, 미국 집으로 배송 주문 완료. → 컨셉의 핵심 가치 실증.

## 시나리오 B — 지원의 목적형 여정 (대안 경로: 카테고리 + 품절 처리)
> 특정 굿즈를 콕 집어 찾는 사용자. 품절이라는 **예외 상황**을 다룬다.

| # | 사용자 행동 | 화면 | 시스템 / AI 반응 | 관련 데이터 |
|---|------------|------|-----------------|------------|
| 1 | 헤더 **카테고리 → 키링** 클릭 | 카테고리 목록 | category=`KEYRING` 상품만 필터 | Product.category |
| 2 | 원하던 키링 발견, 클릭 | S3 | 상세 표시. 그런데 **stock=0** | Product.stock |
| 3 | "품절" 상태 확인 | S3 | 담기 버튼 비활성 + "Sold out" 표기 (2차: 재입고 알림) | is_active/stock |
| 4 | 챗봇에 "비슷한 키링 있어?" | G1 | AI가 **같은 카테고리 재고 있는 상품** 추천 | Product |
| 5 | 대체 상품 담기 → 주문 | S3~S6 | 시나리오 A의 6~12와 동일 | Cart/Order |

**포인트**: 목적형 탐색(카테고리)과 대화형(챗봇)이 만나 **품절이라도 이탈하지 않게** 한다.

## 시나리오 C — Kenji의 원격 구매 여정 (글로벌: 배송·통화 확인 우선)
> 물건보다 "해외로 오나? 총 얼마?"가 먼저인 사용자.

| # | 사용자 행동 | 화면 | 시스템 / AI 반응 | 관련 데이터 |
|---|------------|------|-----------------|------------|
| 1 | 진입 후 언어를 **EN**(또는 일본어는 2차)로 | G2 | UI 전환 | i18n |
| 2 | 챗봇에 "Do you ship to Japan? How much?" | G1 | AI가 **해외배송 가능 + region=ASIA 배송비** 안내 | shipping_region |
| 3 | 안심하고 팝업 둘러보기 | S1→S2 | 팝업/굿즈 탐색 | Popup, Product |
| 4 | 상품 담고 체크아웃, 국가=Japan | S5 | region=`ASIA` 배송비 + **총액(₩ 및 ≈$)** 미리 표시 | 환율, 배송비 |
| 5 | 모의결제 → 완료 | S6 | 주문 생성 | Order |

**포인트**: **배송 가능 여부·총액을 주문 전에 명확히** 보여주는 것이 원격 구매자의 신뢰를 만든다.

## 부분 시나리오 (짧은 흐름들)
- **D1 · 언어 전환**: 어느 화면에서든 G2로 KO↔EN 즉시 전환, 현재 페이지 유지.
- **D2 · 챗봇 지름길**: "recommend a cute ecobag under 15000" → AI가 조건 맞는 상품 카드로.
- **D3 · 빈 장바구니**: S4에서 장바구니가 비면 "둘러보기로" 유도 링크.
- **D4 · 팝업 마감**: status=`CLOSED` 팝업은 카드에 'Ended' 배지, 상세는 열람만/구매 제한.

## 예외·경계 상황 체크리스트 (개발 시 잊지 말 것)
| 상황 | 기대 동작 |
|------|----------|
| 재고 0 | 담기 비활성 + "품절/Sold out" 표기 |
| 배송 불가 국가(2차) | 안내 메시지, 주문 차단 |
| 필수 입력 누락(이름/이메일/주소) | 체크아웃 진행 불가 + 안내 |
| 언어 전환 시 | 담은 장바구니·현재 위치 유지 |
| 챗봇이 못 찾음 | "다시 말해줄래?" 또는 인기 팝업 대체 제안 |
| 가격 표시 | 항상 KRW 기준 + USD 참고 환산 동시 노출 |

## 이 시나리오가 검증하는 것 (성공기준 연결 · `01` 참고)
- **학습 성공**: 시나리오 A의 1~12가 프론트→백엔드→DB까지 완주하면 = 목표 달성.
- **비즈니스 가설**: 외국인(P1·P3)이 A/C를 끝까지 "사고 싶다"고 느끼는가 → 5명 시연 검증.
- **UX 가설**: 탐색형(A)·목적형(B)·원격형(C)이 모두 막힘없이 주문에 도달하는가.
