# 성수동 글로벌 굿즈 쇼핑몰 — 기획 발굴 "십문십답" 🎯

> 기획 발굴(product discovery) 산출물. 기획 전문 서브 에이전트가 생성한, 이 제품을
> 어떻게 만들지 스스로 정리하도록 돕는 핵심 질문 10개 + 초심자용 추천 답안(초안).
> 사용자가 각 질문에 답(동의/수정)하면 이 문서가 곧 1페이지 기획서가 된다.
>
> **활용법**: 각 질문의 💡 추천 답안을 출발점 삼아 "동의하나? 다르게 갈까?"를 메모.
> 10개를 다 채우면 화면 목록 → 데이터 → 개발 할 일로 이어진다.

---

### Q1. 우리의 진짜 고객은 누구인가? (타겟 고객)
- **왜 중요한가:** 고객이 누구냐에 따라 언어·결제·디자인·배송이 전부 달라진다. "모두를 위한 사이트"는 "아무를 위한 것도 아닌 사이트"가 되기 쉽다.
- **생각할 거리:**
  - 성수동 팝업을 찾는 **20~30대 외국인 관광객**(K-컬처 팬, 일본·중국·동남아·서구권)
  - 팝업을 즐기는 **국내 20~30대 트렌드세터**(SNS 인증 세대)
  - 한국에 못 오지만 성수동 팝업 굿즈를 갖고 싶어하는 **해외 원격 구매자**
  - 비중을 정해보기: 예) 외국인 관광객 60% : 내국인 40%
- **💡 추천 답안(초안):** **1순위 = 한국을 방문 중/예정인 20~30대 외국인 관광객, 2순위 = 국내 트렌드세터.** → "글로벌"이 핵심이므로, 외국인이 겪는 불편(언어·결제·해외배송)을 푸는 쪽에 차별점이 있다.

### Q2. 왜 하필 이 사이트에서 사야 하는가? (핵심 가치 & 차별점)
- **왜 중요한가:** 고객에겐 이미 무신사·쿠팡·팝업 현장이라는 선택지가 있다. "여기서만 되는 것"이 없으면 오지 않는다.
- **생각할 거리:**
  - 현장 구매 단점 해결: **줄 서기 / 품절 / 캐리어에 못 넣음 / 무거워서 못 삼**
  - 외국인의 벽 해결: **한국어뿐이라 못 삼 / 해외배송 안 됨 / 결제 실패**
  - 정보 가치: **"지금 성수동에 어떤 팝업이 열리는지" 한눈에 큐레이션**
- **💡 추천 답안(초안):** **"성수동 팝업 굿즈를, 줄 안 서고 외국어로, 우리 집(해외 포함)까지." + 지금 열리는 팝업 큐레이션.** → 현장 3대 불편(대기·품절·짐)과 외국인 3대 벽(언어·결제·배송)을 동시에 없애는 것이 가장 뾰족한 차별점.

### Q3. 사이트가 성수동 팝업과 어떻게 연결되는가? ("팝업 연결" 메커니즘)
- **왜 중요한가:** 이 부분이 쇼핑몰의 "정체성". 그냥 굿즈몰과 "성수동 팝업 굿즈몰"을 가르는 핵심 장치.
- **생각할 거리:**
  - **팝업 단위 큐레이션:** 상품을 브랜드가 아닌 "OO 팝업" 페이지로 묶어 전시
  - **팝업 일정 연동:** "이번 주 성수동 팝업 지도/캘린더"(열림·마감 표시)
  - **현장 QR 연결:** 팝업 현장 QR → 해당 팝업 굿즈 페이지로 바로 이동
  - **품절/한정 알림:** 현장 매진 상품을 온라인 예약·재입고 알림
- **💡 추천 답안(초안):** **"팝업 단위 큐레이션" + "성수동 팝업 캘린더"를 1차 핵심으로, QR 연결은 2차로.** → 팝업별 페이지 구조는 난이도가 낮으면서 컨셉을 가장 확실히 살린다.

### Q4. 어떤 굿즈를 파는가? (판매 범위)
- **왜 중요한가:** 취급 범위가 좁으면 초기 개발·재고가 쉽고, 넓으면 복잡. 초심자는 **좁게 시작**이 유리.
- **생각할 거리:**
  - **특정 팝업 공식 굿즈만**(예: 한 시즌 3~5개 팝업 상품)
  - **편집숍형**(여러 팝업/브랜드 폭넓게)
  - 카테고리 예시: **키링·스티커·에코백·티셔츠·아크릴 스탠드·포토카드** 등 가볍고 배송 쉬운 것
- **💡 추천 답안(초안):** **"동시에 열리는 2~4개 팝업의 공식 굿즈 중, 가볍고 부피 작은 품목(키링·스티커·에코백류) 위주".** → 해외배송비가 저렴하고 파손 위험이 낮아 "글로벌"과 궁합이 좋고 재고 관리도 감당 가능.

### Q5. 굿즈를 어떻게 확보·보관·발송하는가? (소싱/재고)
- **왜 중요한가:** 물건이 오가는 방식은 비즈니스의 현실. 단, **학습 프로젝트에서는 "진짜 재고 없이"** 흐름만 구현해도 된다.
- **생각할 거리:**
  - **직접 매입 재고**(통제 쉬움, 돈·공간 필요)
  - **위탁/드롭십**(재고 부담 없음, 통제 약함)
  - **제휴 입점**(브랜드가 상품·재고 관리, 나는 플랫폼)
  - 학습용: **가상 재고 데이터(더미)로 주문→재고차감→배송상태** 구현
- **💡 추천 답안(초안):** **기획서상 "팝업 브랜드 제휴 + 위탁 발송" 모델, 실제 개발은 가상 재고 DB로 전체 흐름 구현.** → 재고 부담 없는 제휴가 현실적이고, 학습 목표는 더미 데이터로 충분히 달성.

### Q6. "글로벌"을 어디까지 지원하는가? (다국어·통화·해외배송)
- **왜 중요한가:** 글로벌은 무한히 넓힐 수 있어, 범위를 안 정하면 끝없이 는다. **1차 출시에서 어디까지만** 할지 선을 그어야 한다.
- **생각할 거리:**
  - 언어: **한국어+영어(2개국어)로 시작** vs 일·중까지 4개국어
  - 통화: **원화(KRW) 기본 + 화면 환산 표시** vs 실제 다중통화 결제
  - 배송: **국내배송만 우선** vs 처음부터 해외배송(국가·배송비 계산)
- **💡 추천 답안(초안):** **MVP는 "한/영 2개국어 / KRW 기준가 + USD 참고 환산 / 배송은 국가 선택 + 고정 권역 배송비" 수준.** → 실제 국제결제·관세 계산은 뒤로, "외국인이 이해하고 주문 시도까지" 되는 경험을 먼저 완성.

### Q7. 이 몰에 자연스러운 AI 기능은 무엇인가? (AI 주인공 기능)
- **왜 중요한가:** 이 과정의 핵심은 AI 접목. "있어 보이려는 AI"가 아니라 **이 컨셉에서 진짜 쓸모 있는 AI**여야 한다.
- **생각할 거리:**
  - **다국어 AI 챗봇:** 외국인이 영어로 "지금 갈 만한 성수동 팝업?" 물으면 추천
  - **취향 기반 굿즈 추천:** "미니멀/캐릭터/아트" 취향 → 맞춤 큐레이션
  - **자동 번역:** 브랜드가 올린 한글 상품 설명 → 영어 자동 번역
  - **AI 굿즈 설명 생성:** 상품 정보 입력 시 매력적 소개문 자동 작성
- **💡 추천 답안(초안):** **"다국어 AI 쇼핑 도우미 챗봇"을 주인공, "상품설명 자동 번역"을 보조로.** → 외국인의 최대 벽이 언어인데, 챗봇 하나로 "팝업 추천+굿즈 추천+질문 응대"를 다국어로 묶으면 컨셉·타겟·AI가 한 방향.

### Q8. 1차 출시에 꼭 넣을 것 vs 미룰 것은? (MVP 범위)
- **왜 중요한가:** 초심자가 가장 많이 실패하는 이유는 "다 만들려다 아무것도 못 끝내서". **작게 끝내는 것**이 실력.
- **생각할 거리:**
  - 반드시(MVP): **상품목록 / 상품상세 / 장바구니 / 주문(모의결제) / 팝업별 큐레이션 / AI 챗봇 기본형**
  - 나중에: 실제 결제(PG)·회원등급·리뷰·위시리스트·재입고 알림·정교한 해외배송비
- **💡 추천 답안(초안):** **MVP = 「팝업별 굿즈 큐레이션 + 상품상세 + 장바구니 + 모의 주문 + 한/영 전환 + AI 챗봇 1개」까지만.** → 이 6가지면 "쇼핑몰 한 바퀴"가 돌아 프론트·백·데이터·AI를 전부 경험, 나머지는 뒤에 붙인다.

### Q9. 어떻게 돈을 버는가? (비즈니스 모델)
- **왜 중요한가:** 학습 프로젝트여도 "사업이라면 어떻게 유지되나"를 설계해야 진짜 기획을 배운다. 수익 구조가 화면(가격·정산)에도 영향.
- **생각할 거리:**
  - **직접판매 마진**(통제 강, 재고 부담)
  - **제휴 수수료**(판매액의 %)
  - **입점 수수료**(월/건당)
  - 부가: 팝업 브랜드 대상 **홍보·상단 노출 광고**
- **💡 추천 답안(초안):** **"판매 제휴 수수료(마켓플레이스형)"를 기본 모델로.** → 재고를 직접 안 사도 되어 리스크가 낮고, "여러 팝업 굿즈를 모아 파는 큐레이션 플랫폼" 컨셉과 잘 맞는다.

### Q10. 무엇으로 "성공"을 판단하고, 무엇을 먼저 검증하는가? (성공 기준 & 첫 검증)
- **왜 중요한가:** 기준이 없으면 "잘 되고 있는지"를 알 수 없다. 초심자는 **가장 위험한 가정 하나를 먼저 확인**하는 습관이 중요.
- **생각할 거리:**
  - 학습 성공 기준: **"주문 한 건이 프론트→백엔드→DB까지 완주" / AI 챗봇이 실제 답변**
  - 비즈니스 가설: **"외국인은 정말 성수동 팝업 굿즈를 온라인에서 사고 싶어할까?"**
  - 검증 방법: 지인·외국인 친구 5명 시연 → **"이거 있으면 쓸래?" 피드백**
  - 지표 예: 방문자 중 장바구니 담기 비율, 챗봇 사용률
- **💡 추천 답안(초안):** **학습 성공 = "모의 주문 1건 + 다국어 챗봇 응답 끝까지 작동", 비즈니스 첫 검증 = "외국인 5명에게 보여주고 '사고 싶다' 반응".** → 거창한 매출보다 핵심 가설을 가장 싸고 빠르게 확인하는 것이 초기 성공.
