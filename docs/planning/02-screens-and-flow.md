# 02 · 화면 목록 & 사용자 흐름 (Screens & Flow)  — v1.0

> 기획 2단계 산출물. 컨셉(v1.0)의 MVP 6기능을 **실제 만들 화면 단위**로 옮긴 것.
> 이 목록이 곧 개발 할 일 목록의 뼈대가 된다. 각 화면 옆에 필요한 핵심 데이터도
> 미리 적어 두어 다음 단계(데이터 모델)로 자연스럽게 연결한다.

## 사용자 흐름 (구매 여정)
```
[홈: 팝업 큐레이션]
   │  (지금 열리는 팝업들을 본다)
   ▼
[팝업 상세: 해당 팝업의 굿즈 목록]
   │  (마음에 드는 굿즈 클릭)
   ▼
[상품 상세]  ──담기──▶ [장바구니]
   │                      │  (배송 국가 선택)
   │                      ▼
   │                 [주문/체크아웃] (비회원, 모의결제)
   │                      ▼
   │                 [주문 완료]
   │
   └───────── 어느 화면에서든 ─────────┐
                                         ▼
                            [AI 챗봇] (전역 위젯, 다국어)
   ※ 언어 전환(한/영)·통화 표시(KRW+USD)는 모든 화면 공통(전역)
```

## MVP 화면 목록
| # | 화면 | 핵심 역할 | 필요한 주요 데이터 |
|---|------|----------|-------------------|
| S1 | **홈 (팝업 큐레이션)** | 지금 열리는 성수동 팝업 목록/캘린더 | 팝업(이름·기간·썸네일·상태) |
| S2 | **팝업 상세** | 특정 팝업 소개 + 그 팝업의 굿즈 목록 | 팝업 정보 + 소속 상품 목록 |
| S3 | **상품 상세** | 굿즈 정보·가격·설명, 장바구니 담기 | 상품(이름·가격·이미지·설명·재고) |
| S4 | **장바구니** | 담은 상품 확인·수량 변경·합계 | 장바구니 항목(상품·수량) |
| S5 | **주문/체크아웃** | 배송지(국가)·배송비·모의결제 | 주문자정보·배송국가·배송비·합계 |
| S6 | **주문 완료** | 주문 번호·요약 확인 | 생성된 주문(번호·내역·상태) |
| G1 | **AI 챗봇** (전역 위젯) | 다국어 팝업/굿즈 추천·질문 응대 | 팝업·상품 데이터 + 대화 |
| G2 | **언어/통화 전환** (전역) | 한↔영, KRW+USD 환산 표시 | i18n 문자열·환율(고정값) |

## 화면별 메모 (초심자 관점 포인트)
- **S1 홈**: MVP에서 "캘린더"는 복잡한 달력 UI 대신 **"현재 열림/마감 임박" 배지가 붙은
  팝업 카드 목록**으로 단순화해도 충분하다.
- **S3 상품 상세**: 상품 설명은 한글 원문 + 영어(자동 번역) 두 벌을 보여주는 것이
  AI 보조 기능과 연결된다.
- **S5 체크아웃**: 회원가입 없음(비회원). 배송비는 **권역별 고정값**(예: 국내/아시아/
  기타)으로 단순화. 결제는 "결제하기" 클릭 시 실제 승인 없이 주문 생성(모의).
- **G1 챗봇**: MVP 1개 챗봇. 상품·팝업 데이터를 컨텍스트로 넣어 답하게 한다
  (AI 학습의 핵심 지점). 실제 LLM 연동은 개발 단계에서.

## 2차 이후로 미룬 화면 (지금 안 만듦)
로그인/회원가입·마이페이지·주문내역, 리뷰 작성, 위시리스트, 관리자(상품/팝업 등록),
재입고 알림, QR 랜딩.
