## MCP (Model Context Protocol)  (2026-07-02, 기록 체계를 세우며)
- **한 줄 정의**: AI(예: Claude)가 외부 도구·데이터 소스에 표준화된 방식으로 접근하게
  해주는 프로토콜. "AI를 위한 USB-C 포트"에 비유된다.
- **핵심 포인트**:
  - MCP 서버가 파일·DB·API 같은 **컨텍스트(지식)**를 AI에게 제공한다.
  - 우리가 남기는 `docs/`의 잘 구조화된 MD 문서들이 나중에 MCP 서버의 **지식 소스**가
    될 수 있다.
- **우리 프로젝트에 적용**: 지금은 문서를 MCP가 파싱하기 좋게 원자적·일관된 형식으로
  남겨두는 것이 핵심. 실제 MCP 서버화는 문서가 어느 정도 쌓인 뒤 별도 단계로 진행.

## 데이터 모델링 핵심 3원칙  (2026-07-02, 데이터 모델 초안을 만들며)
- **한 줄 정의**: 무엇을 어떤 표(엔티티)로, 어떤 열(필드)로, 어떻게 연결(관계)해 저장할지의 설계.
- **핵심 포인트**:
  - **스냅샷 원칙**: 주문처럼 "그 시점의 사실"은 값을 복사해 박제한다. 원본(상품가격)이
    바뀌어도 과거 기록이 흔들리지 않게. → OrderItem에 이름·가격 복사.
  - **저장 대신 계산**: 팝업 열림/마감처럼 시간이 지나면 낡는 값은 저장하지 말고
    그때그때 계산(오늘 날짜 vs 기간).
  - **과설계 금지(YAGNI)**: 다국어를 당장 `_ko`/`_en` 두 열로 단순하게. 필요해지면 그때 확장.
- **적용**: MVP 엔티티 = Popup·Product·CartItem·Order·OrderItem. User/결제/리뷰는 뒤로.

## 정보구조(IA)  (2026-07-02, UX 개선용 IA를 만들며)
- **한 줄 정의**: 콘텐츠를 어떻게 조직하고(구조), 사용자가 어떻게 찾아다니게 할지(탐색)의 설계.
  "정보의 지도와 이정표".
- **핵심 포인트**: 4요소 = 사이트맵(구조) · 내비게이션(이동) · 라벨링(표지판) · 탐색 경로(길).
  조직 축을 잘 고르는 게 UX의 절반 — 우리는 1차 '팝업별', 2차 '카테고리별'.
- **적용**: 탐색형·목적형·대화형(챗봇) 세 갈래 길을 상품 상세로 수렴시켜 다양한 사용자 커버.

## Next.js 서버/클라이언트 컴포넌트  (2026-07-02, 화면 개발하며)
- **한 줄 정의**: App Router에서 컴포넌트는 기본 '서버'에서 렌더(빠르고 DB 직접 접근 가능),
  상호작용이 필요하면 파일 맨 위 `"use client"`로 '클라이언트' 컴포넌트로 만든다.
- **핵심 포인트**: 데이터 조회(상품 목록)는 서버 컴포넌트에서, 버튼 클릭·장바구니 같은
  상태는 클라이언트 컴포넌트에서. 우리는 카드·상세는 서버, 헤더·장바구니·챗봇은 클라이언트.
- **적용**: 로케일은 서버가 쿠키로 읽어 콘텐츠 렌더, 언어 토글(클라이언트)이 쿠키+`router.refresh()`.

## Prisma 드라이버 어댑터 (Prisma 7)  (2026-07-02, DB 연결하며)
- **한 줄 정의**: Prisma 7부터 DB 연결을 '드라이버 어댑터'로 붙인다. SQLite는
  `@prisma/adapter-better-sqlite3`를 `new PrismaClient({ adapter })`로 주입.
- **핵심 포인트**: 예전처럼 `new PrismaClient()`만으로는 에러. 어댑터가 실제 DB 드라이버 역할.
  스키마의 `datasource`에 `url=env(...)`를 두면 런타임이 연결 문자열을 읽는다.
- **적용**: `src/lib/db.ts`에서 어댑터로 클라이언트 생성, 개발 중 인스턴스 폭증 막게 전역 캐싱.

## AI에 '컨텍스트 주입'으로 환각 막기  (2026-07-02, 챗봇 만들며)
- **한 줄 정의**: LLM이 실제 데이터에 근거해 답하도록, 프롬프트(system)에 우리 DB의
  팝업·상품 목록을 텍스트로 넣어준다.
- **핵심 포인트**: "목록에 없는 상품은 지어내지 마"라고 지시 + 실데이터 제공 → 없는 상품
  추천(환각) 방지. 사용자 언어에 맞춰 답하게 지시.
- **적용**: `/api/chat`에서 `getCatalogForContext()`로 카탈로그를 만들어 system 프롬프트에 주입.
